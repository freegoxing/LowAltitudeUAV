# 低空无人机通信与应急救援相关数据集调研报告

在“低空复杂环境下任务导向自适应Token级智能语义通信与规划系统”的研发中，很难找到一个直接包含“大模型决策 + 语义通信 + 链路自愈 + 物理路径规划”所有维度的单一现成数据集。

通常的学术与工程做法是**“分层使用现有数据集”**或**“使用仿真器合成数据集”**。以下是针对本系统不同层级所整理的开源数据集和仿真资源：

---

## 1. 物理链路与信道层数据集（用于信道预测、SNR建模与遮挡分析）

这类数据集提供无人机在低空（A2G、A2A）传输时的真实/仿真物理层指标（如接收信号强度 RSSI、信噪比 SNR、时延扩展等），可用于训练底层的图注意力网络或强化学习环境。

| 数据集名称 | 发布时间/平台 | 数据内容与特点 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **AERPAW A2G Channel Datasets** | [AERPAW 官网](https://aerpaw.org/) | 包含真实的空地（A2G）射频信号测量数据（SigMF 格式）。记录了无人机在不同高度（40m、70m、100m）、不同频段（如 3.3GHz）下的信噪比、路径损耗及信道冲击响应（CIR），并附带 GPS Telemetry 数据和 Python 处理脚本。 | **链路质量评估**：用于建模无人机运动时的动态 SNR 衰减。 |
| **Multi-Modal Benchmark Dataset for UAV Wireless Comm** | 2026 (MDPI) | 融合了三维几何结构、传播条件分类（LOS/NLOS）以及链路统计数据的多模态基准数据集。支持城市、郊区和农村场景。 | **三维路径规划**：根据障碍物几何体预测非视距（NLOS）遮挡与信号中断。 |
| **Geometry-Aware Propagation Prediction Dataset** | IEEE DataPort | 包含高精度射线追踪（Ray-tracing）生成的城市低空信道数据。包含大量无人机轨迹上的路径损耗和多径参数。 | **图神经网络（GNN）训练**：用作 Graph Transformer 提取拓扑特征的地面真值。 |

---

## 2. 网络拓扑与链路故障数据集（用于 FANET 路由与自愈切换验证）

这类数据集聚焦于网络层协议（如 OLSR、AODV），记录了多节点移动时拓扑频繁分裂、合并以及链路中断时的丢包率、吞吐量和时延。

| 数据集名称 | 发布时间/平台 | 数据内容与特点 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **FANET Dataset (NS-3.40)** | 2026 (ResearchGate) | 基于 NS-3.40 仿真生成的飞行自组网数据集。包含 8 种系统级场景下的数据包追踪、节点移动轨迹、剩余能量指标以及 OLSR 路由表状态。 | **链路自愈与切换**：验证在某些中继节点突发断链或电量耗尽时，备份路由的切换性能。 |
| **UAV Network Communication Dataset** | Kaggle | 记录了不同无人机密度、信道竞争窗口调整策略下的吞吐量、丢包率和网络负载数据。 | **资源分配策略**：用于 Agent 2 进行通信资源倾斜（带宽分配）的基准对比。 |

---

## 3. 语义感知与灾害场景数据集（用于前端语义提取与目标定位）

这类数据集属于计算机视觉/语义地图领域，提供无人机视角的图像/视频及像素级标注，可用于机载端提取“语义 Token”回传。

| 数据集名称 | 核心关注点 | 标注类型与类别 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **RescueNet** | **防灾与应急救援** | 高分辨率航拍图像的像素级语义分割。类别包括：建筑物受损程度（轻度/重度/完全坍塌）、道路状况（畅通/被阻断）、水体、车辆及植被。 | **搜救任务语义理解**：评估受灾区域，规划救援路径时识别避障与畅通路段。 |
| **VisDrone** | **目标检测与人群计数** | 包含大量城市场景下的低空动态视频。标注了行人、车辆、自行车等多种移动目标。 | **人员搜救定位**：训练无人机机载模型识别地面救援人员（GND-P）及救援目标（TGT）。 |
| **UAVid** | **城市倾斜视角感知** | 侧重于城市场景的语义分割，包含动态对象（人类、车辆）和静态背景。 | **低空三维避障**：构建语义知识图谱中的静态障碍物节点属性。 |

---

## 4. 本项目的数据集构建方案推荐（合成/仿真方案）

由于学术界开源的数据集通常是**静态**的（例如只提供某段轨迹的 SNR，或只提供受灾后的图片），无法与大模型智能体进行**动态交互**。为了验证本系统，推荐采用**“半物理仿真/合成数据集”**的生成管线：

### 推荐工具链组合
1. **网络与信道生成：NS-3 / MATLAB**
   * 使用 3GPP TR 38.901 信道模型，根据无人机实时三维坐标计算动态 SNR 和时延。
   * 通过 NS-3 模拟多跳中继（FANET），设置突发节点下线（模拟链路中断），导出拓扑变化日志作为**网络拓扑 Token 序列**。
2. **环境与运动仿真：AirSim / Webots / Gazebo**
   * 导入高程图（DEM）或受灾城市三维模型。
   * 运行强化学习算法（PPO/SAC），生成无人机避障飞行轨迹。
3. **大模型决策数据：Synthetic LLM Interaction Log**
   * 使用 GPT-4 或 Claude 模拟 Agent 1 和 Agent 2 的输入输出，生成包含 `[环境状态, 紧急度评估, 奖励权重 W, 资源倾斜指令]` 的训练对，用于微调本地轻量化大模型。

### 总结
* 如果你想做**信道与物理路径规划**的验证，推荐使用 **AERPAW** 或 **Multi-Modal Benchmark (2026)** 的信道数据。
* 如果你想做**救援场景的语义识别**，推荐使用 **RescueNet**。
* 如果要验证**多路径自愈算法**，最标准的方法是利用 **NS-3.40** 仿真环境注入故障来获取实时拓扑数据。
