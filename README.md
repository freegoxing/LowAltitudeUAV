# 低空无人机语义通信与规划系统 (Low-Altitude UAV Semantic Communication and Planning System)

本项目构建了一个**面向低空复杂环境的任务导向自适应Token级智能语义通信与规划系统**。针对无人机在低空执行救援、巡检等任务时面临的复杂物理与通信环境（如遮挡、干扰、动态变化等），本项目通过构建语义通信知识图谱并结合大模型（LLMs）进行任务导向的智能规划与调度。

## 系统方法论 (Methodology)

传统系统通常将“信息传输”与“任务决策”割裂。本系统通过以下核心层实现了“认知”与“动作”的有效解耦和深度融合：

1. **知识图谱建模层（感知基座）：** 将低空环境（气象、障碍、基站等）提取并统一转化为语义图谱（Knowledge Graph），映射为 Token 流。
2. **双 LLM 智能体协同层：** 
   - **Agent 1（态势评估）：** 负责判断任务的紧急度和可行性。
   - **Agent 2（语义任务下发）：** 将抽象任务语义翻译为底层网络控制可执行的参数（如奖励函数权重）。
3. **图感知驱动的强化学习规划层：** 结合 Graph Transformer 和强化学习算法（PPO/SAC），通过动态调整的目标奖励参数进行无人机的自适应路径规划。

详细方法论请参考：[低空无人机系统方法论文档](./docs/low_altitude_uav_methodology.md)

## 场景规划设计 (Scene Planning & Design)

系统中定义了多元化的**通信主体节点**及**链路关系边**：

- **空域节点：** 任务无人机 (`UAV-M`)、中继无人机 (`UAV-R`)、侦察无人机 (`UAV-S`)。
- **地面节点：** 救援人员终端 (`GND-P`)、地面指挥车 (`GND-C`)、固定基站 (`BS`)。
- **逻辑标记：** 目标信息源 `(TGT)`。
- **关系边（链路）：** 空-空链路 (A2A)、空-地链路 (A2G)、地-地链路 (G2G)、中继链路 (RLY) 等。

通过构建多路径通信链路规划与自愈机制（“主路径 + N条备份路径 + 热备份节点”），系统能够动态保障恶劣环境下的通信可靠性。

详细场景设计请参考：[场景规划设计文档](./docs/scene_planning_design.md)

## 当前代码实现：基于 SNR 的状态建模

> **核心提示**：在目前的早期代码实现中，系统主要**聚焦于使用信噪比 (SNR, Signal-to-Noise Ratio) 作为核心的状态指标**来进行环境建模与路径评估。

无论是在数据构造（如构建 `mock_uav_network.json` 中的 `snr_uplink` 和 `snr_downlink`）还是在链路连通性判断中，当前版本的逻辑将 SNR 作为节点间相互作用、建立通信边以及后续智能决策的主要依据。后续版本中将逐步融入多模态传感器数据、能耗、时延等多维度的语义状态。
